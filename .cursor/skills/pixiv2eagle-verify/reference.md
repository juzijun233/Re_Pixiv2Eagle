# Pixiv2Eagle 手测详细步骤

## 1. 插画保存

**页面**：`https://www.pixiv.net/artworks/{id}`（单页或多页插画）

1. 确认「保存到 Eagle」按钮出现在作品详情页
2. 点击保存，观察 toast 反馈
3. 在 Eagle 中确认：文件入库、文件夹层级（画师 → 类型「插画」）、`item.url` 为 `https://www.pixiv.net/artworks/{id}`
4. 多页作品：若开启「为多页作品创建子文件夹」，确认子文件夹与分页文件
5. 推荐区域：已保存作品缩略图应出现 ✅ 标记

## 2. ugoira 动图

**页面**：动图作品页（`illustType === 2`）

1. 保存动图作品
2. 确认 CDN 库（JSZip / fflate / gif.js）加载无报错
3. Eagle 中应为 GIF 格式，文件夹归类在「插画」类型下

## 3. 漫画系列

**页面**：`https://www.pixiv.net/user/{uid}/series/{seriesId}`

1. 确认「更新系列序号」按钮出现
2. 点击更新，观察 toast 与 Eagle 文件夹重命名/序号更新
3. 系列页列表上已保存章节应有 ✅ 标记

## 4. 画师列表标记

**页面**：`https://www.pixiv.net/users/{uid}/artworks` 或 `/manga`

1. 确认列表缩略图上已保存作品显示 ✅
2. 向下滚动加载更多，新加载项也应正确标记
3. 若改动选择器，对比 `config/selectors/list.js`

## 5. 小说三格式

**页面**：`https://www.pixiv.net/novel/show.php?id={id}`

1. 菜单切换保存格式：TXT → MD → EPUB，各保存一次
2. 确认 Eagle 中文件格式与内容正确
3. EPUB 需确认 CDN（JSZip）加载正常
4. 已保存状态 UI（按钮文案/样式）应正确反映

## 6. 小说系列

**页面**：`https://www.pixiv.net/novel/series/{seriesId}`

1. 确认系列章节列表上已保存章节有 ✅ 标记
2. 未保存章节无错误标记

## 7. 菜单项

Tampermonkey 脚本菜单（点击扩展图标 → 本脚本）：

| 菜单项 | 验证要点 |
|--------|---------|
| 📁 设置 Pixiv 文件夹 ID | 输入有效 Eagle 文件夹 ID 后保存 |
| 📅 切换：使用投稿时间 | 切换后保存作品，检查 Eagle 添加日期 |
| 🕗 切换：保存作品描述 | 切换后保存，检查 item 描述 |
| 🗂️ 切换：多页子文件夹 | 切换后保存多页作品 |
| 🗂️ 切换：按类型保存 | 切换后确认类型文件夹创建 |
| 🖼️ 保存当前作品 | 在当前作品页触发保存 |
| 🔎 切换：自动检测保存状态 | 开关后刷新页面，标记行为变化 |
| 🔄 强制更新 Eagle 索引 | 执行后列表标记应刷新 |
| 📂 设置小说保存路径 | 设置本地路径后保存小说 |
| 📚 切换：小说保存格式 | 见第 5 项 |
| 🧪 调试模式 | 开启后控制台有 `[Pixiv2Eagle]` 日志 |
| 🧪 设置画师文件夹名称模板 | 修改模板后新建画师文件夹命名变化 |
